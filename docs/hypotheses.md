# Hypotheses — Smart Onboarding

## Hypothesis 1 — Guided first steps
We believe that a personalised onboarding checklist will help Christoph understand what to do first because it presents clear tasks in a logical order.

[Feasibility: Low effort — static checklist UI, no backend needed. Strong Week 3 slice candidate.]

## Hypothesis 2 — Progress visibility
We believe that showing onboarding progress will help Christoph feel more confident because he can see completed tasks, remaining tasks, and the next recommended action.

[Feasibility: Low-Medium — progress bar/tracker, can use simple local state. Good Week 3 slice candidate.]
## Hypothesis 3 — Centralised information
We believe that a single onboarding hub will help Christoph find essential documents, contacts, and resources faster because important information is organised in one place.

[Feasibility: Medium-High — needs a real content/document structure behind it. Too big for Week 3; mock with static sample data instead.]
## Hypothesis 4 — Contextual support
We believe that providing help links and suggested contacts within each onboarding task will help Christoph resolve questions more easily because support is available at the moment it is needed.

[Feasibility: Low-Medium — static help/contact list tied to each task. Buildable as a small component.]
## Hypothesis 5 — Flexible onboarding
We believe that allowing Christoph to revisit, skip, or save tasks for later will help him manage onboarding at his own pace because workplace priorities can change during the first weeks.

[Feasibility: Medium — needs state to track skipped/saved tasks and revisit logic. Doable but more complex than 1, 2, or 4.]
